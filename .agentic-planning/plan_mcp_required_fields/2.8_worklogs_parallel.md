# 2.8 Worklogs API - Добавление обязательного `fields` (Parallel)

**Тип:** Parallel | **Приоритет:** Высокий | **Tools:** 3

---

## Tools

1. **add_worklog** - добавление записи времени (возвращает Worklog)
2. **get_worklogs** - список записей (возвращает Worklog[])
3. **update_worklog** - обновление записи (возвращает Worklog)

**Не требует:** `delete_worklog` (void)

---

## Изменения

### add_worklog
- **Schema:** `src/tools/api/worklog/add/add-worklog.schema.ts` → `fields: FieldsSchema`
- **Tool:** `src/tools/api/worklog/add/add-worklog.tool.ts` → `ResponseFieldFilter.filter<WorklogWithUnknownFields>(worklog, fields)`
- **Tests:** Добавить `fields: ['id', 'duration', 'comment']`

### get_worklogs
- **Schema:** `src/tools/api/worklog/get/get-worklogs.schema.ts` → `fields: FieldsSchema`
- **Tool:** `src/tools/api/worklog/get/get-worklogs.tool.ts` → `worklogs.map(w => ResponseFieldFilter.filter(w, fields))`
- **Tests:** Добавить `fields`

### update_worklog
- **Schema:** `src/tools/api/worklog/update/update-worklog.schema.ts` → `fields: FieldsSchema`
- **Tool:** `src/tools/api/worklog/update/update-worklog.tool.ts` → `ResponseFieldFilter.filter(worklog, fields)`
- **Tests:** Добавить `fields`
